# Создание тестового аккаунта подрядчика (Contractor)

## Описание
Этот скрипт создает тестового пользователя с ролью `contractor` для проверки функционала страницы подрядчика.

## Что делает скрипт
1. Создает профиль пользователя с ролью `contractor`
2. Настраивает необходимые данные для отображения на странице подрядчика

## Пошаговая инструкция

### Шаг 0: Выполнение миграции базы данных (ОБЯЗАТЕЛЬНО!)

⚠️ **ВАЖНО**: Перед созданием пользователя-подрядчика необходимо добавить роль `contractor` в базу данных.

1. Откройте **Supabase SQL Editor**
2. Откройте файл `supabase/add_contractor_role.sql`
3. Скопируйте весь SQL код из файла
4. Вставьте в SQL Editor
5. Нажмите **Run** для выполнения
6. Дождитесь успешного выполнения (должно появиться сообщение "Success. No rows returned")

**Что делает эта миграция:**
- Обновляет CHECK constraint в таблице `profiles`
- Добавляет роль `contractor` к разрешенным значениям (client, organizer, main_organizer, contractor)

### Шаг 1: Создание пользователя в Supabase Authentication

1. Откройте Supabase Dashboard
2. Перейдите в **Authentication** → **Users**
3. Нажмите **Add User** → **Create new user**
4. Заполните данные:
   - **Email**: `contractor@test.com`
   - **Password**: `Test123!` (или любой другой пароль не менее 6 символов)
   - **Auto Confirm User**: Включите (чтобы не нужно было подтверждать email)
5. Нажмите **Create User**
6. **ВАЖНО**: Скопируйте **User ID** созданного пользователя (обычно это UUID формата `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`)

### Шаг 2: Выполнение SQL скрипта

1. Откройте файл `scripts/create_contractor_user.sql`
2. Замените `YOUR_CONTRACTOR_USER_ID` на реальный User ID из Шага 1
3. Если использовали другой email, замените `contractor@test.com` на ваш email
4. Откройте **Supabase SQL Editor**
5. Вставьте отредактированный SQL код
6. Нажмите **Run** для выполнения

### Шаг 3: Проверка

1. Раскомментируйте проверочный запрос в конце SQL скрипта
2. Выполните запрос в SQL Editor
3. Убедитесь, что пользователь создан с ролью `contractor`

### Шаг 4: Вход в систему

1. Откройте приложение
2. Перейдите на страницу входа `/login`
3. Введите credentials:
   - **Email**: `contractor@test.com`
   - **Password**: `Test123!` (или пароль, который вы установили)
4. После входа вы будете автоматически перенаправлены на `/contractor`

## Данные на странице подрядчика

Страница подрядчика отображает следующую информацию о событии:
- **Дата** - дата события
- **Место проведения** - локация события
- **Количество гостей** - ожидаемое число гостей
- **Имена пары** - имена молодоженов
- **Дресс-код** - требования к одежде
- **Тайминг** - расписание события
- **Контакты организатора** - контактная информация координатора
